Congress Data Dashboard — Project Overview

🚧 Current Project Status

This project is currently running as a manually refreshed data pipeline and dashboard. Originally, the system was designed for nightly automated processing of congressional data updates.

During development, a more complete and reliable legislative dataset was identified late in the project timeline. Because of that discovery, the automated nightly processing component is temporarily paused while the pipeline is being redesigned around the improved data source.

Future updates will include:

  • Automated nightly ETL processing
  • Incremental bill updates
  • Expanded congressional datasets
  • Improved machine learning predictions and analytics
  • More advanced visualizations and trend tracking

The current version still demonstrates the full architecture and workflow of the system, including XML parsing, ETL processing, DuckDB integration, analytics generation, and D3.js visualization.

🧠 Overview

This project builds a data pipeline and interactive dashboard that tracks and analyzes U.S. congressional bills. It processes raw legislative data, structures it into a database, computes activity metrics, and visualizes the results in a web dashboard.

The goal of the system is to create a scalable platform for monitoring legislative activity and exploring congressional trends through interactive analytics.

⚙️ Data Pipeline (ETL Process)

1. Extract

Raw XML files containing congressional bill records are parsed. Data pulled from Gov Info Each file includes:

  • Bill metadata (number, title, type, chamber, introduced date)
  • Legislative actions (steps the bill has gone through)
  • Co-sponsor information

2. Transform

The extracted data is cleaned and structured into three datasets:

Bills → core bill information (ID, title, sponsor, type)
Actions → full legislative timeline
Co-sponsors → supporting legislators

During this stage, the system also computes:

  • Total actions per bill
  • Total co-sponsors per bill
  • Normalized identifiers for joining datasets

3. Load

The cleaned data is stored in a local analytical database (DuckDB). A final JSON file is generated:

congress_dashboard.json

This file powers all visualizations in the web dashboard.

📊 Dashboard Layer

The dashboard is a browser-based application that reads the JSON output and provides interactive exploration of congressional activity.

📋 Bill Explorer Table

  • Bill number and type
  • Title
  • Sponsor
  • Action count
  • Co-sponsor count

Users can:

  • Sort by any column
  • Search across bills, sponsors, and titles
  • Paginate through results

🔍 Detail View

Clicking a bill opens a detailed modal showing:

  • Sponsor information
  • Full co-sponsor list
  • Complete legislative action timeline

📈 Key Metrics

  • Total bills processed
  • Total legislative actions
  • Total co-sponsors
  • Average actions per bill

These metrics provide a high-level summary of legislative activity and system-wide trends.

🧩 Design Concept

The system is designed around a simple principle:

Transform raw legislative data → structured dataset → interactive insights

It combines:

  • Data engineering (ETL pipeline)
  • Database design (DuckDB)
  • Analytics (aggregation and feature engineering)
  • Frontend visualization (D3.js dashboard)
  • Machine Learning Items

🤖 Machine Learning Components

In addition to the ETL pipeline and dashboard visualizations, the project also includes machine learning and analytical modeling components designed to explore legislative behavior and bill progression patterns.

🔮 Legislative Next-Step Prediction

One machine learning component focuses on predicting the likely next legislative step for a congressional bill based on historical activity and bill characteristics.

The prediction models analyze features such as:

  • Bill type
  • Legislative chamber
  • Sponsor information
  • Action history
  • Number of co-sponsors
  • Current bill status

Multiple machine learning classification models were tested and compared to evaluate prediction performance and confidence levels.

🧩 Congressional Bill Clustering

A second analytical component applies clustering techniques to group similar congressional bills together based on shared legislative characteristics and activity patterns.

The clustering process evaluates:

  • Legislative activity levels
  • Action counts
  • Co-sponsor participation
  • Bill progression behavior
  • Structural similarities between bills

These clusters help identify patterns such as:

  • Highly active legislation
  • Low-engagement bills
  • Similar legislative progression trends
  • Groups of bills with related behavioral characteristics

The clustering visualization provides an additional analytical layer beyond traditional reporting by revealing hidden relationships between congressional bills.
